BARBEARIA OS V16 — ASSISTENTE IA INTERNO

O QUE ENTROU
- Nova página admin-assistente.html
- Indicadores do dia, receita prevista, pendências, clientes inativos, aniversários e ausências
- Perguntas rápidas
- Campo de conversa com a JuIA
- Edge Function segura: ju-ia-admin
- Nenhuma chave secreta fica no GitHub ou no navegador

1) GITHUB
Publique todos os arquivos do ZIP, substituindo os atuais.
Commit sugerido: Barbearia OS V16 - Assistente IA interno

2) SQL
Execute atualizacao-barbearia-os-v16.sql no SQL Editor do Supabase.
Resultado esperado: Success. No rows returned

3) EDGE FUNCTION
Supabase > Edge Functions > Deploy a new function > Via Editor.
Nome: ju-ia-admin
Cole o conteúdo de supabase/functions/ju-ia-admin/index.ts.
IMPORTANTE: desative a verificação JWT automática / escolha a opção equivalente a "No verify JWT", pois o código já valida a sessão internamente e o projeto usa a nova publishable key.
Clique em Deploy function.

4) SEGREDO OPENAI
Supabase > Edge Functions > Secrets.
Crie:
OPENAI_API_KEY = sua chave secreta da API OpenAI
Nunca coloque essa chave no GitHub, HTML ou JavaScript público.
O modelo configurado é gpt-5.6-luna, escolhido para equilibrar qualidade e custo.

5) TESTE
Abra:
https://www.barbeariadoju.com.br/admin-assistente.html?v=16

Sem OPENAI_API_KEY, a página funciona em modo de análise local.
Com a chave configurada, o status muda para "IA ativa" e as respostas passam pela OpenAI.

OBSERVAÇÃO
Esta V16 é a IA interna do painel. O webhook do WhatsApp será uma função separada, criada depois que esta base estiver testada.
